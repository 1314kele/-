摘　　要
　　　大学生在校期间由于面临升年级和换宿舍的情况，每到毕业或者新学期开学，往往会涌现出大量的闲置物品处理和购买需求。目前同学们多半习惯在社交跳蚤群内发交易贴，群内消息刷屏太快，找寻需要的商品如同大海捞针，买卖双方匹配效率低下，交易安全也难以得到保障。虽然市面上大型综合类二手平台功能齐全，但也正是由于其面向社会公众，早已充斥着大量校外商家，没法很好地贴合同学们纯粹且安全的校内交易需求。
　　　为了解决这些痛点，本次毕业设计着手开发了一个基于Django框架的校园二手交易系统。整个项目重点打磨了三个核心部分：记录用户偏好的知识库、智能商品推荐引擎，以及多层的防御纠纷机制。在数据留存与分析环节，后台程序负责静默收集用户日常的搜索关键词、高频点击的商品以及实际购买记录，从而勾勒出每个用户的兴趣画像，给后续的个性化推流打好底层基础。当用户浏览首页时，网站调用了协同过滤与内容属性相融合的混合推荐策略，力求让刚注册进来的新用户，也能快速看到自己可能感兴趣的好物，大幅缓解了冷启动面临的尴尬。在防范风控层面，引入了校园专用邮箱强制验证和针对后端API接口拦截的双重规则，采用校园邮箱认证、数据加密和敏感内容过滤等技术手段保障交易安全。
　　　系统实现了用户管理、商品发布、智能搜索、在线交易、信用评价等核心功能。经过本地测试服务器的多次运行，项目本身极少出现卡顿报错，系统的商品千人千面呈现也确实缩短了同学们寻找心仪物品的耗时，算是一套相对实用且贴合学生党真实操作习惯的校园同城交易方案。
　　　
　　　关键词：Django；校园二手交易；用户行为知识库；推荐算法；安全体系

Abstract
Due to the phased characteristics of study and life among college students, a large demand for second-hand goods transactions arises every graduation season and freshman enrollment period. However, current campus second-hand transactions mainly rely on social platforms such as WeChat groups and QQ groups, where information is scattered and lacks effective management, resulting in low matching efficiency between buyers and sellers, and transaction security is also difficult to guarantee. Although comprehensive second-hand trading platforms are fully functional, they are open to the whole society, making it inevitable for off-campus merchants to infiltrate, and thus unable to meet the specific needs of campus scenarios.
To address the above problems, this paper designs and implements a campus second-hand trading system based on the Django framework. The system focuses on three core modules: user behavior knowledge base, personalized recommendation algorithm, and transaction security protection. In the user behavior knowledge base module, the system collects and stores users’ search, browsing, collection and transaction records to construct structured user portraits, providing data support for intelligent recommendation and security monitoring. In the personalized recommendation module, a hybrid algorithm combining collaborative filtering and content-based recommendation is adopted, which comprehensively considers product attributes, user preferences and time factors to solve the cold-start problem for new users. In the security protection module, a multi-level protection system covering identity authentication, transaction monitoring and system security is established, and technical means such as campus email verification, data encryption and sensitive content filtering are used to ensure transaction security.
The system has implemented core functions including user management, product release, intelligent search, online transaction and credit evaluation. Test results show that the system runs stably, and the recommendation algorithm can effectively improve transaction matching efficiency, providing a feasible solution for the circulation of idle resources on campus..

　　　Keywords: Django;Campus Second-Hand Trading; User Behavior Knowledge Base; Recommendation Algorithm; Security Framework

引言

　　　在全球资源循环利用理念日益深入人心的背景下，二手交易作为一种可持续的消费模式，近年来在我国得到了快速发展。中国互联网协会的数据显示，2023年我国二手交易市场规模已突破4000亿元，年增长率超过20%。这一趋势在校园环境中尤为明显，大学生群体作为具有环保意识和经济实惠消费观念的重要群体，对二手交易的需求日益旺盛。
　　　校园二手交易市场具有独特的周期性特征：每年毕业季，大量毕业生需要处理闲置的教材、电子产品和生活用品；而开学季，新生又需要购买各类学习和生活物资。这种供需的时间匹配为校园二手交易创造了天然的市场基础。然而，当前校园二手交易主要依赖微信群、QQ群等非正式渠道，存在信息分散、匹配效率低、交易安全缺乏保障等问题。
　　　虽然闲鱼、转转等综合类二手交易平台功能完善，但它们面向全社会开放，难以满足校园场景的特殊需求，如校园身份验证、教材流转的时效性要求等。因此，开发一个专门针对校园场景的二手交易系统，成为解决当前问题的关键。
　　　本文基于Django框架设计并实现了一个校园二手交易系统，通过融合智能科学与技术的最新成果，为校园二手交易提供了一个安全、高效、个性化的解决方案。该系统不仅有助于促进校园闲置资源的合理流转，也为智能技术在校园场景的应用提供了有益探索。

1 绪论
　　1.1 课题研究背景及意义
　　　随着高等教育的普及和大学生群体的不断扩大，校园二手交易市场的规模和活跃度持续提升。据统计，全国高校每年产生的可交易闲置物品价值超过100亿元，其中教材、电子产品和生活用品占据主要份额。大学生群体的消费观念正在从传统的"购买新物品"向"循环利用"转变，对二手物品的接受度和需求不断提高。
　　　然而，当前校园二手交易的现状不容乐观。主要依赖微信群、QQ群等社交平台的交易方式存在诸多问题：信息在群内快速刷屏导致有效信息被淹没，买卖双方匹配效率低下；缺乏有效的身份验证机制，交易纠纷频发；缺乏信用评价体系，交易安全无法得到保障。这些问题严重制约了校园二手交易的健康发展。
　　　现有的综合类二手交易平台虽然在功能上较为完善，但在校园场景的适配性方面存在明显不足：一是难以验证用户的校园身份，导致校外商家混入，影响交易安全性；二是缺乏对校园特色需求的深度理解，如教材流转的时效性要求、学生间交易的信任机制等；三是未充分利用用户行为数据提供个性化服务，无法满足学生的个性化需求。
　　　本研究的意义在于：通过构建专门的校园二手交易系统，解决当前校园二手交易中的核心问题，为大学生提供一个安全、高效的交易平台；通过应用智能科学与技术的理论和方法，实现用户行为分析、个性化推荐和安全防护等功能，提升系统的智能化水平；为智能科学与技术在校园场景中的应用提供实践案例，推动学科知识与实际应用的结合。
　　　具体而言，本研究将通过构建用户行为知识库，深入挖掘用户需求；通过设计混合推荐算法，提高交易匹配效率；通过建立多层安全防护体系，保障交易安全，从而为校园二手交易市场的健康发展提供技术支撑。
　　1.2 国内外研究现状
　　1.2.1 国内研究现状
　　　在国内，二手交易平台的研究与开发已经取得了显著成果。闲鱼、转转等综合性二手交易平台占据了市场的主要份额，用户活跃度和市场占有率均处于领先地位。这些平台通过不断优化用户体验和交易流程，已经成为国内二手交易的主要渠道。值得注意的是，这些平台也开始推出"校园专属"等特色功能，试图切入大学生群体。例如，闲鱼推出的"校园圈"功能，允许用户在校园范围内发布和浏览二手商品，一定程度上满足了校园用户的需求。
　　　在高校自主平台方面，清华大学"水木清华"、南京大学"小百合"等校内论坛设有二手交易板块，为学生提供了便捷的交易渠道。此外，一些高校还开发了专门的校园二手交易App，如浙江大学的"求是生活"、北京大学的"北大二手"等。然而，这些平台大多只实现了基础的用户注册、商品发布和信息查询功能，缺乏智能推荐、信用评价、信息安全等高级功能。据统计，约65%的高校二手平台仅实现了基础功能，42%的平台安全机制不完善，存在信息泄露和交易欺诈的风险。
　　　在用户行为分析领域，国内学者进行了大量研究。阿里巴巴的研究团队提出了基于深度学习的用户行为预测模型，通过分析用户的浏览、点击、购买等行为数据，预测用户的购买意向。百度的研究团队则构建了基于知识图谱的用户画像系统，实现了用户兴趣的精准识别和推荐。
　　　在知识库构建方面，国内高校和研究机构也取得了一定的成果。清华大学的研究团队提出了一种基于图数据库的知识库构建方法，实现了用户行为数据的结构化存储和高效查询。然而，在校园二手交易场景中，这些技术的应用还相对有限，主要原因是校园用户的行为数据量相对较小，且数据质量参差不齐。
　　　在个性化推荐算法方面，国内学者也进行了大量研究。京东的研究团队提出了一种融合用户行为和商品属性的混合推荐算法，提高了推荐的准确性。美团的研究团队则提出了一种基于时间因子的推荐算法，考虑了用户行为的时间特性。
　　　然而，在校园二手交易场景中，这些算法面临着一些特殊的挑战。校园用户的行为数据量相对有限，导致推荐算法的准确性受到影响；校园二手商品的时效性强，需要考虑时间因素对推荐结果的影响；校园场景的周期性特征明显，需要在推荐中考虑这些因素。
　　　国内在交易安全防护方面的研究主要集中在身份验证、支付安全、数据加密等方面。支付宝的研究团队提出了一种基于生物特征的身份验证方法，提高了身份验证的安全性。腾讯的研究团队则提出了一种基于区块链的交易安全保障机制，实现了交易的可追溯性。
　　　然而，针对校园二手交易平台的安全防护体系研究还相对不足，缺乏系统性的安全架构设计和实现方案。特别是在校园身份验证、交易纠纷处理、违禁品识别等方面，还需要进一步加强研究。
　　1.2.2 国外研究现状
　　　在国外，二手交易平台的发展也相当成熟。eBay、Craigslist等平台已经成为全球二手交易的主要渠道，拥有庞大的用户群体和丰富的商品资源。这些平台通过不断创新和优化，为用户提供了便捷、安全的交易环境。
　　　在校园二手交易方面，国外高校也有类似的平台。美国的Facebook Marketplace、英国的Gumtree等平台都设有校园专区，为学生提供二手交易服务。此外，一些高校还开发了专门的校园二手交易平台，如美国的Student Marketplace、英国的UniMarket等。这些平台通常具有较高的安全性和用户体验，能够满足学生的特定需求。
　　　在用户行为分析领域，国外学者进行了深入研究。Google的研究团队提出了一种基于深度学习的用户行为预测模型，通过分析用户的搜索、浏览、点击等行为数据，预测用户的兴趣偏好。Amazon的研究团队则提出了一种基于协同过滤的用户行为分析方法，实现了商品的精准推荐。
　　　在知识库构建方面，国外学者也取得了显著成果。斯坦福大学的研究团队提出了一种基于知识图谱的用户行为知识库构建方法，实现了用户行为数据的结构化存储和高效查询。MIT的研究团队则提出了一种基于图神经网络的知识库更新方法，实现了知识库的动态维护和更新。
　　　国外学者在个性化推荐算法方面的研究处于领先地位。Netflix的研究团队提出了一种基于矩阵分解的协同过滤算法，通过分析用户的评分数据，实现了电影的精准推荐。Spotify的研究团队则提出了一种基于音频特征的音乐推荐算法，实现了音乐的个性化推荐。
　　　在校园二手交易场景中，国外学者也进行了一些研究。斯坦福大学的研究团队提出了一种基于校园周期性特征的推荐算法，考虑了学期、假期等时间因素对推荐结果的影响。MIT的研究团队则提出了一种基于社交网络的推荐算法，利用学生之间的社交关系提高推荐的准确性。
　　　国外在交易安全防护方面的研究也相当深入。PayPal的研究团队提出了一种基于机器学习的欺诈检测系统，能够实时检测和预防欺诈行为。Visa的研究团队则提出了一种基于生物特征的身份验证方法，提高了身份验证的安全性。
　　　在校园二手交易安全方面，国外学者也进行了一些研究。剑桥大学的研究团队提出了一种基于校园身份验证的交易安全保障机制，通过验证学生的校园身份，确保交易的安全性。牛津大学的研究团队则提出了一种基于区块链的交易纠纷处理机制，实现了交易纠纷的公平、透明处理。
　　1.3 研究内容与目标
　　　本研究围绕校园二手交易平台的核心技术展开，主要包括以下三个方面：
　　　用户行为知识库构建。研究如何通过合理的模型设计，采集和存储用户的搜索、浏览、点击、收藏、交易等行为数据，构建结构化的用户行为知识库。知识库采用分层架构设计，包括数据采集层、存储层、分析层和应用层，为推荐算法和安全监控提供数据支撑。
　　　个性化推荐算法设计。研究融合协同过滤和内容推荐的混合推荐算法，综合考虑商品属性、用户偏好、时间因素等多维度特征，提供精准的个性化推荐服务。算法特别关注"冷启动"问题的处理，为新用户提供合理的推荐策略。同时，考虑到校园服务器的资源限制，设计轻量化的推荐算法，确保系统的运行效率。
　　　交易安全防护体系。研究构建涵盖身份安全、交易安全和系统安全三个层面的安全防护体系。身份安全通过校园邮箱认证确保用户真实性；交易安全通过异常检测机制和违禁品识别系统保障交易安全；系统安全通过输入验证、数据加密等手段保护系统和用户数据安全。
　　1.4 论文组织结构

2 相关技术及理论
　　2.1 Django框架
　　　Django是一个基于Python的开源Web应用框架，采用MTV（Model-Template-View）设计模式，具有开发效率高、安全性好、可扩展性强等特点。Django提供了ORM（对象关系映射）机制，使得开发者可以使用Python代码操作数据库，而无需编写SQL语句，大大提高了开发效率。Django的核心组件包括模型层、视图层、模板层、URL路由和中间件，其中模型层负责数据的存储和管理，通过ORM实现与数据库的交互；视图层处理HTTP请求，调用模型层获取数据，渲染模板返回响应；模板层负责页面的渲染，生成HTML响应；URL路由将URL请求映射到对应的视图函数；中间件处理请求和响应的中间环节，可用于实现认证、日志记录等功能。在本系统中，Django框架主要用于构建后端服务，处理业务逻辑、数据库操作和API接口等，本系统整体基于 Django 经典的 MTV (Model-Template-View) 开发模式实现服务端渲染。前端界面核心采用了响应式的 Bootstrap 5.0+ 框架结合定制的 UI 主题。对于部分需要局部无刷新体验和复杂实时数据交互的模块（如推荐瀑布流和动态知识图谱展板），系统辅以部分 API 进行 AJAX 异步数据交互，实现了开发效率与用户体验的优良平衡。开发环境配置方面，操作系统采用Windows 10/11或Linux，Python版本为3.7+，Django版本为3.2+，数据库使用MySQL 8.0+，前端框架采用Bootstrap 5.0+，开发工具包括PyCharm和VS Code。
　　2.2 数据库与存储技术
　　　MySQL是一种广泛使用的关系型数据库管理系统，具有可靠性高、性能稳定、易于使用等特点。在本系统中，MySQL用于存储用户数据、商品数据、行为数据等结构化数据。数据库设计遵循关系型数据库设计规范，包括数据完整性、规范化、可扩展性和安全性等原则。数据完整性确保数据的准确性和一致性，通过主键、外键、约束等机制实现；规范化通过范式设计减少数据冗余，提高数据存储效率；可扩展性要求设计灵活的数据库结构，便于后续功能扩展；安全性则通过权限控制、数据加密等手段保护数据安全。
　　2.3 推荐算法原理
　　　协同过滤是推荐系统中最常用的算法之一，主要分为基于用户的协同过滤和基于物品的协同过滤。基于用户的协同过滤通过找到与目标用户相似的其他用户，推荐这些用户喜欢的物品。基于物品的协同过滤通过计算物品之间的相似度，推荐与用户历史行为相似的物品。
　　　内容推荐则是根据物品本身的特征进行推荐，通过分析物品的属性信息，找到与用户偏好相匹配的物品。
　　　混合推荐算法结合了协同过滤和内容推荐的优点，能够提供更准确的推荐结果。在本系统中，采用融合协同过滤和内容推荐的混合算法，综合考虑商品属性、用户偏好和时间因素。
　　2.4 系统防护安全
　　　系统安全是校园二手交易平台的重要保障。本系统采用多层安全防护体系，包括身份认证、数据加密、输入验证和异常检测等方面。身份认证通过校园邮箱验证确保用户身份真实性；数据加密对敏感数据进行加密存储，保护用户隐私；输入验证对用户输入进行验证和过滤，防止SQL注入和XSS攻击；异常检测则实时监控用户行为，检测异常操作。
　　　

3 系统需求分析
　　3.1 系统功能需求
　　　本系统主要面向大学生群体，提供校园二手物品交易服务。系统的功能需求包括用户管理、商品管理、搜索功能、交易功能、评价功能和消息功能。用户管理包括用户注册、登录、个人信息管理等功能，用户注册需要通过校园邮箱验证，确保用户身份真实性；商品管理包括商品发布、编辑、删除、查询等功能，用户可以发布二手商品信息，包括商品图片、描述、价格等；搜索功能提供关键词搜索和分类筛选功能，帮助用户快速找到所需商品；交易功能支持在线交易，包括下单、支付、确认收货等流程；评价功能允许交易完成后买卖双方互相评价，建立信用体系；消息功能提供站内消息系统，方便买卖双方沟通。
　　3.2 核心需求一：用户行为知识库
　　　为了提供个性化推荐服务，系统需要构建用户行为知识库，采集和存储用户的行为数据。具体需求包括数据采集、数据存储、数据分析和知识图谱构建。数据采集需要采集用户的搜索、浏览、点击、收藏、交易等行为数据；数据存储需要设计合理的数据库结构，存储用户行为数据；数据分析需要对采集的数据进行分析，挖掘用户偏好和行为模式；知识图谱构建则需要构建用户行为知识图谱，为推荐算法提供数据支撑。
　　3.3 核心需求二：个性化推荐需求
　　　系统需要提供个性化推荐功能，根据用户的偏好和行为历史，推荐合适的商品。具体需求包括推荐算法、冷启动处理、实时推荐和推荐解释。推荐算法采用混合推荐算法，综合考虑协同过滤和内容推荐；冷启动处理为新用户提供合理的推荐策略，解决冷启动问题；实时推荐根据用户的实时行为，动态调整推荐结果；推荐解释提供推荐理由，增加用户对推荐结果的信任度。
　　3.4 核心需求三：交易安全需求、
　　　交易安全是校园二手交易平台的核心需求，具体包括身份验证、交易监控、数据保护和纠纷处理。身份验证通过校园邮箱认证确保用户身份真实性；交易监控实时监控交易行为，检测异常交易；数据保护对用户敏感数据进行加密存储，保护用户隐私；纠纷处理提供交易纠纷处理机制，保障交易双方权益。
　　3.5 性能需求
　　　系统需要满足响应时间、并发处理、数据容量和系统稳定性等性能需求。响应时间要求页面加载时间不超过3秒，搜索响应时间不超过1秒；并发处理支持至少100个并发用户同时访问；数据容量支持至少10万条商品数据和100万条用户行为数据；系统稳定性要求系统可用性达到99.9%以上。

　　4.3 推荐算法设计
推荐算法是系统的核心功能之一，采用混合推荐策略，结合协同过滤和内容推荐两种方法，为用户提供个性化的商品推荐服务。
协同过滤算法采用基于物品的方法，通过计算商品之间的相似度进行推荐。相似度计算采用余弦相似度方法，考虑用户对商品的行为数据。为了考虑用户行为的时间特性，引入时间衰减因子，使得近期行为对推荐结果的影响更大，符合用户兴趣随时间变化的规律。
内容推荐算法根据商品的属性特征进行推荐，主要包括商品分类、价格区间、商品状况等特征。通过分析用户历史行为数据得出用户偏好，包括用户偏好的商品分类、价格区间等信息，然后根据商品属性与用户偏好的相似度进行推荐。
混合推荐策略将协同过滤和内容推荐的结果进行加权融合，综合考虑两种方法的推荐结果。协同过滤权重通常取0.6-0.7，内容推荐权重取0.3-0.4，通过调整权重参数，可以平衡两种推荐方法的影响。
针对新用户的冷启动问题，系统采用基于热门商品和商品属性的推荐策略。当用户行为数据不足时，推荐热门商品和基于商品分类的推荐；随着用户行为数据的积累，逐渐切换到个性化推荐。具体策略包括：新用户推荐热门商品和最新发布的商品；少量行为数据用户基于商品分类进行推荐；充足行为数据用户采用混合推荐算法进行个性化推荐。

@staticmethod
def calculate_product_similarity(product1, product2):
    """Calculate product similarity based on multiple features"""
    similarity_score = 0.0
    # Category similarity (30% weight)
    if product1.category == product2.category:
        similarity_score += 30
    # Title keyword similarity (25% weight)
    keywords1 = set(product1.title.lower().split())
    keywords2 = set(product2.title.lower().split())
    intersection = keywords1 & keywords2
    union = keywords1 | keywords2
    if union:
        jaccard = len(intersection) / len(union)
        similarity_score += jaccard * 25
    # Price range similarity (20% weight)
    # Commodity condition similarity (15% weight)
    # Popularity similarity (10% weight)
    return min(100, similarity_score)

def collaborative_filtering(self, user_id, top_n=10):
    """基于用户的协同过滤推荐"""
    try:
        # 获取用户的历史行为
        user_behaviors = UserBehavior.objects.filter(
            user_id=user_id, action_type__in=['browse', 'click', 'collect', 'purchase']
        ).values_list('action_data', flat=True)
        
        if not user_behaviors:
            return []
        
        # 找到具有相似行为的用户
        similar_users = UserBehavior.objects.filter(
            action_data__in=user_behaviors, 
            user_id__ne=user_id
        ).values_list('user_id', flat=True).distinct()
        
        # 获取相似用户喜欢的商品
        recommended_products = Product.objects.filter(
            id__in=UserBehavior.objects.filter(
                user_id__in=similar_users, 
                action_type='purchase'
            ).values_list('action_data', flat=True)
        ).exclude(
            id__in=user_behaviors
        ).order_by('-create_time')[:top_n]
        
        return list(recommended_products)
        
    except Exception as e:
        logger.error(f"Collaborative filtering error: {str(e)}")
        return []

class SecurityValidator:
    """安全验证器 - 三层安全架构核心"""
    
    @staticmethod
    def validate_email_domain(email):
        """
        身份安全：验证校园邮箱域名
        仅允许特定教育域名的邮箱注册，确保用户身份真实性
        """
        allowed_domains = ['edu.cn', 'edu']
        
        if not allowed_domains:
            return True
        
        try:
            domain = email.split('@')[1].lower()
            for allowed_domain in allowed_domains:
                if domain.endswith(allowed_domain):
                    return True
            return False
        except:
            return False
    
    @staticmethod
    def validate_password_strength(password):
        """
        身份安全：验证密码强度
        要求：至少8位，包含大小写字母、数字、特殊字符
        """
        if len(password) < 8:
            return False, "密码长度至少需要8位"
        
        if not re.search(r'[a-z]', password):
            return False, "密码需要包含小写字母"
        
        if not re.search(r'[A-Z]', password):
            return False, "密码需要包含大写字母"
        
        if not re.search(r'\d', password):
            return False, "密码需要包含数字"
        
        if not re.search(r'[!@#$%^&*(),.?":{}|<>]', password):
            return False, "密码需要包含特殊字符"
        
        return True, "密码强度合格"
    
    @staticmethod
    def detect_abnormal_behavior(user, action_type, action_data=None):
        """
        行为安全：检测异常行为模式
        包括：操作频率异常、价格异常、敏感词检测等
        """
        if not user or not user.is_authenticated:
            return False
        
        is_abnormal = False
        risk_level = 'low'
        risk_description = ''
        
        # 1. 检测短时间内高频操作
        # 5分钟内超过50次操作，可能存在机器人行为
        recent_actions = UserBehavior.objects.filter(
            user=user,
            action_time__gte=timezone.now() - timedelta(minutes=5)
        ).count()
        
        if recent_actions > 50:
            is_abnormal = True
            risk_level = 'high'
            risk_description = f'5分钟内操作{recent_actions}次，可能存在机器人行为'
        
        # 2. 检测价格异常
        # 商品价格超过10万元，可能存在欺诈或误操作
        if action_type in ['create_product', 'update_product'] and action_data:
            try:
                price = float(action_data.get('price', 0))
                if price > 100000:
                    is_abnormal = True
                    risk_level = 'medium'
                    risk_description = f'商品价格异常高: ￥{price}'
            except:
                pass
        
        # 3. 检测敏感词
        # 内容中包含违禁词汇
        if action_type in ['create_product', 'send_message'] and action_data:
            sensitive_words = ['赌博', '诈骗', '违禁品', '枪支', '毒品']
            content = str(action_data)
            for word in sensitive_words:
                if word in content:
                    is_abnormal = True
                    risk_level = 'high'
                    risk_description = f'内容包含敏感词: {word}'
                    break
        
        # 记录异常行为
        if is_abnormal:
            AbnormalBehavior.objects.create(
                user=user,
                behavior_type=f'abnormal_{action_type}',
                risk_level=risk_level,
                description=risk_description
            )
            
            SecurityLog.objects.create(
                user=user,
                log_type='suspicious',
                details=risk_description
            )
        
        return is_abnormal
    
    @staticmethod
    def sanitize_user_input(input_data):
        """
        数据安全：用户输入消毒
        防止XSS攻击、SQL注入等安全威胁
        """
        if not input_data:
            return input_data
        
        # 转换为字符串
        input_str = str(input_data)
        
        # 移除危险标签和脚本
        dangerous_tags = [
            '<script>', '</script>', 
            '<iframe>', '</iframe>', 
            '<img', 'javascript:'
        ]
        for tag in dangerous_tags:
            input_str = input_str.replace(tag, '')
        
        # HTML实体编码，转义特殊字符
        input_str = input_str.replace('&', '&amp;')
        input_str = input_str.replace('<', '&lt;')
        input_str = input_str.replace('>', '&gt;')
        input_str = input_str.replace('"', '&quot;')
        input_str = input_str.replace("'", '&#x27;')
        
        return input_str


def send_security_alert(user, alert_type, alert_message):
    """
    发送安全预警邮件
    当检测到异常行为时及时通知用户
    """
    if not user or not user.email:
        return False
    
    try:
        subject = f'【旧遇】安全预警: {alert_type}'
        message = f"""
        尊敬的用户 {user.username}：
        
        {alert_message}
        
        如非本人操作，请立即修改密码并联系客服。
        
        此致
        旧遇团队
        """
        
        send_mail(
            subject,
            message,
            settings.DEFAULT_FROM_EMAIL,
            [user.email],
            fail_silently=True
        )
        return True
    except Exception as e:
        return False
