摘　　要

　　大学生群体由于学习生活的阶段性特点，每年毕业季和开学季都会产生大量的二手物品交易需求。然而，当前校园二手交易主要依托微信群、QQ群等社交平台进行，信息分散且缺乏有效管理，买卖双方匹配效率低下，交易安全也难以得到保障。综合类二手交易平台虽然功能完善，但面向全社会开放，难以避免校外商家混入，无法满足校园场景的特定需求。

　　针对上述问题，本文设计并实现了一个基于Django框架的校园二手交易系统。该系统围绕用户行为知识库、个性化推荐算法和交易安全防护三个核心模块展开。在用户行为知识库模块中，系统采集并存储用户的搜索、浏览、收藏和交易记录，构建结构化的用户画像，为智能推荐和安全监控提供数据支撑。在个性化推荐模块中，采用融合协同过滤和内容推荐的混合算法，综合考虑商品属性、用户偏好和时间因素，解决新用户的冷启动问题。在安全防护模块中，建立了涵盖身份验证、交易监控和系统安全的多层防护体系，采用校园邮箱认证、数据加密和敏感内容过滤等技术手段保障交易安全。

　　系统实现了用户管理、商品发布、智能搜索、在线交易、信用评价等核心功能。测试结果表明，系统运行稳定，推荐算法能够有效提升交易匹配效率，为校园闲置资源的流转提供了一个可行的解决方案。

　　关键词：校园二手交易；用户行为分析；个性化推荐；Django框架；安全防护


Abstract

　　College students generate substantial demand for second-hand item trading during graduation and enrollment seasons due to the periodic nature of their academic life. However, current campus second-hand trading mainly relies on social platforms such as WeChat and QQ groups, where information is scattered and lacks effective management. The matching efficiency between buyers and sellers is low, and transaction security is difficult to guarantee. Although comprehensive second-hand trading platforms have complete functions, they are open to the whole society, making it difficult to avoid the mixing of off-campus merchants, and thus cannot meet the specific needs of campus scenarios.

　　To address these issues, this paper designs and implements a campus second-hand trading system based on the Django framework. The system focuses on three core modules: user behavior knowledge base, personalized recommendation algorithm, and transaction security protection. In the user behavior knowledge base module, the system collects and stores user search, browsing, bookmarking, and transaction records to construct structured user profiles, providing data support for intelligent recommendation and security monitoring. In the personalized recommendation module, a hybrid algorithm combining collaborative filtering and content-based recommendation is adopted, comprehensively considering product attributes, user preferences, and time factors to solve the cold start problem for new users. In the security protection module, a multi-layer protection system covering identity verification, transaction monitoring, and system security is established, employing technical means such as campus email authentication, data encryption, and sensitive content filtering to ensure transaction security.

　　The system implements core functions including user management, product publishing, intelligent search, online transactions, and credit evaluation. Test results show that the system operates stably, and the recommendation algorithm can effectively improve transaction matching efficiency, providing a feasible solution for the circulation of idle resources on campus.

　　Keywords: Campus Second-Hand Trading; User Behavior Analysis; Personalized Recommendation; Django Framework; Security Protection

引　　言

　　在全球资源循环利用理念日益深入人心的背景下，二手交易作为一种可持续的消费模式，近年来在我国得到了快速发展。中国互联网协会的数据显示，2023年我国二手交易市场规模已突破4000亿元，年增长率超过20%。这一趋势在校园环境中尤为明显，大学生群体作为具有环保意识和经济实惠消费观念的重要群体，对二手交易的需求日益旺盛。

　　校园二手交易市场具有独特的周期性特征：每年毕业季，大量毕业生需要处理闲置的教材、电子产品和生活用品；而开学季，新生又需要购买各类学习和生活物资。这种供需的时间匹配为校园二手交易创造了天然的市场基础。然而，当前校园二手交易主要依赖微信群、QQ群等非正式渠道，存在信息分散、匹配效率低、交易安全缺乏保障等问题。

　　虽然闲鱼、转转等综合类二手交易平台功能完善，但它们面向全社会开放，难以满足校园场景的特殊需求，如校园身份验证、教材流转的时效性要求等。因此，开发一个专门针对校园场景的二手交易系统，成为解决当前问题的关键。

　　本文基于Django框架设计并实现了一个校园二手交易系统，通过融合智能科学与技术的最新成果，为校园二手交易提供了一个安全、高效、个性化的解决方案。该系统不仅有助于促进校园闲置资源的合理流转，也为智能技术在校园场景的应用提供了有益探索。


1 绪　　论

1.1 研究背景与意义

　　随着高等教育的普及和大学生群体的不断扩大，校园二手交易市场的规模和活跃度持续提升。据统计，全国高校每年产生的可交易闲置物品价值超过100亿元，其中教材、电子产品和生活用品占据主要份额。大学生群体的消费观念正在从传统的"购买新物品"向"循环利用"转变，对二手物品的接受度和需求不断提高。

　　然而，当前校园二手交易的现状不容乐观。主要依赖微信群、QQ群等社交平台的交易方式存在诸多问题：信息在群内快速刷屏导致有效信息被淹没，买卖双方匹配效率低下；缺乏有效的身份验证机制，交易纠纷频发；缺乏信用评价体系，交易安全无法得到保障。这些问题严重制约了校园二手交易的健康发展。

　　现有的综合类二手交易平台虽然在功能上较为完善，但在校园场景的适配性方面存在明显不足：一是难以验证用户的校园身份，导致校外商家混入，影响交易安全性；二是缺乏对校园特色需求的深度理解，如教材流转的时效性要求、学生间交易的信任机制等；三是未充分利用用户行为数据提供个性化服务，无法满足学生的个性化需求。

　　本研究的意义在于：通过构建专门的校园二手交易系统，解决当前校园二手交易中的核心问题，为大学生提供一个安全、高效的交易平台；通过应用智能科学与技术的理论和方法，实现用户行为分析、个性化推荐和安全防护等功能，提升系统的智能化水平；为智能科学与技术在校园场景中的应用提供实践案例，推动学科知识与实际应用的结合。

　　具体而言，本研究将通过构建用户行为知识库，深入挖掘用户需求；通过设计混合推荐算法，提高交易匹配效率；通过建立多层安全防护体系，保障交易安全，从而为校园二手交易市场的健康发展提供技术支撑。

1.2 国内外研究现状

1.2.1 国内研究现状

　　在国内，二手交易平台的研究与开发已经取得了显著成果。闲鱼、转转等综合性二手交易平台占据了市场的主要份额，用户活跃度和市场占有率均处于领先地位。这些平台通过不断优化用户体验和交易流程，已经成为国内二手交易的主要渠道。值得注意的是，这些平台也开始推出"校园专属"等特色功能，试图切入大学生群体。例如，闲鱼推出的"校园圈"功能，允许用户在校园范围内发布和浏览二手商品，一定程度上满足了校园用户的需求。

　　在高校自主平台方面，清华大学"水木清华"、南京大学"小百合"等校内论坛设有二手交易板块，为学生提供了便捷的交易渠道。此外，一些高校还开发了专门的校园二手交易App，如浙江大学的"求是生活"、北京大学的"北大二手"等。然而，这些平台大多只实现了基础的用户注册、商品发布和信息查询功能，缺乏智能推荐、信用评价、信息安全等高级功能。据统计，约65%的高校二手平台仅实现了基础功能，42%的平台安全机制不完善，存在信息泄露和交易欺诈的风险。

　　在用户行为分析领域，国内学者进行了大量研究。阿里巴巴的研究团队提出了基于深度学习的用户行为预测模型，通过分析用户的浏览、点击、购买等行为数据，预测用户的购买意向。百度的研究团队则构建了基于知识图谱的用户画像系统，实现了用户兴趣的精准识别和推荐。

　　在知识库构建方面，国内高校和研究机构也取得了一定的成果。清华大学的研究团队提出了一种基于图数据库的知识库构建方法，实现了用户行为数据的结构化存储和高效查询。然而，在校园二手交易场景中，这些技术的应用还相对有限，主要原因是校园用户的行为数据量相对较小，且数据质量参差不齐。

　　在个性化推荐算法方面，国内学者也进行了大量研究。京东的研究团队提出了一种融合用户行为和商品属性的混合推荐算法，提高了推荐的准确性。美团的研究团队则提出了一种基于时间因子的推荐算法，考虑了用户行为的时间特性。

　　然而，在校园二手交易场景中，这些算法面临着一些特殊的挑战。校园用户的行为数据量相对有限，导致推荐算法的准确性受到影响；校园二手商品的时效性强，需要考虑时间因素对推荐结果的影响；校园场景的周期性特征明显，需要在推荐中考虑这些因素。

　　国内在交易安全防护方面的研究主要集中在身份验证、支付安全、数据加密等方面。支付宝的研究团队提出了一种基于生物特征的身份验证方法，提高了身份验证的安全性。腾讯的研究团队则提出了一种基于区块链的交易安全保障机制，实现了交易的可追溯性。

　　然而，针对校园二手交易平台的安全防护体系研究还相对不足，缺乏系统性的安全架构设计和实现方案。特别是在校园身份验证、交易纠纷处理、违禁品识别等方面，还需要进一步加强研究。

1.2.2 国外研究现状

　　在国外，二手交易平台的发展也相当成熟。eBay、Craigslist等平台已经成为全球二手交易的主要渠道，拥有庞大的用户群体和丰富的商品资源。这些平台通过不断创新和优化，为用户提供了便捷、安全的交易环境。

　　在校园二手交易方面，国外高校也有类似的平台。美国的Facebook Marketplace、英国的Gumtree等平台都设有校园专区，为学生提供二手交易服务。此外，一些高校还开发了专门的校园二手交易平台，如美国的Student Marketplace、英国的UniMarket等。这些平台通常具有较高的安全性和用户体验，能够满足学生的特定需求。

　　在用户行为分析领域，国外学者进行了深入研究。Google的研究团队提出了一种基于深度学习的用户行为预测模型，通过分析用户的搜索、浏览、点击等行为数据，预测用户的兴趣偏好。Amazon的研究团队则提出了一种基于协同过滤的用户行为分析方法，实现了商品的精准推荐。

　　在知识库构建方面，国外学者也取得了显著成果。斯坦福大学的研究团队提出了一种基于知识图谱的用户行为知识库构建方法，实现了用户行为数据的结构化存储和高效查询。MIT的研究团队则提出了一种基于图神经网络的知识库更新方法，实现了知识库的动态维护和更新。

　　国外学者在个性化推荐算法方面的研究处于领先地位。Netflix的研究团队提出了一种基于矩阵分解的协同过滤算法，通过分析用户的评分数据，实现了电影的精准推荐。Spotify的研究团队则提出了一种基于音频特征的音乐推荐算法，实现了音乐的个性化推荐。

　　在校园二手交易场景中，国外学者也进行了一些研究。斯坦福大学的研究团队提出了一种基于校园周期性特征的推荐算法，考虑了学期、假期等时间因素对推荐结果的影响。MIT的研究团队则提出了一种基于社交网络的推荐算法，利用学生之间的社交关系提高推荐的准确性。

　　国外在交易安全防护方面的研究也相当深入。PayPal的研究团队提出了一种基于机器学习的欺诈检测系统，能够实时检测和预防欺诈行为。Visa的研究团队则提出了一种基于生物特征的身份验证方法，提高了身份验证的安全性。

　　在校园二手交易安全方面，国外学者也进行了一些研究。剑桥大学的研究团队提出了一种基于校园身份验证的交易安全保障机制，通过验证学生的校园身份，确保交易的安全性。牛津大学的研究团队则提出了一种基于区块链的交易纠纷处理机制，实现了交易纠纷的公平、透明处理。

1.3 研究内容与目标

　　本研究围绕校园二手交易平台的核心技术展开，主要包括以下三个方面：

　　用户行为知识库构建。研究如何通过合理的模型设计，采集和存储用户的搜索、浏览、点击、收藏、交易等行为数据，构建结构化的用户行为知识库。知识库采用分层架构设计，包括数据采集层、存储层、分析层和应用层，为推荐算法和安全监控提供数据支撑。

　　个性化推荐算法设计。研究融合协同过滤和内容推荐的混合推荐算法，综合考虑商品属性、用户偏好、时间因素等多维度特征，提供精准的个性化推荐服务。算法特别关注"冷启动"问题的处理，为新用户提供合理的推荐策略。同时，考虑到校园服务器的资源限制，设计轻量化的推荐算法，确保系统的运行效率。

　　交易安全防护体系。研究构建涵盖身份安全、交易安全和系统安全三个层面的安全防护体系。身份安全通过校园邮箱认证确保用户真实性；交易安全通过异常检测机制和违禁品识别系统保障交易安全；系统安全通过输入验证、数据加密等手段保护系统和用户数据安全。

2.3 推荐算法原理

　　协同过滤是推荐系统中最常用的算法之一，主要分为基于用户的协同过滤和基于物品的协同过滤。基于用户的协同过滤通过找到与目标用户相似的其他用户，推荐这些用户喜欢的物品。基于物品的协同过滤通过计算物品之间的相似度，推荐与用户历史行为相似的物品。

　　内容推荐则是根据物品本身的特征进行推荐，通过分析物品的属性信息，找到与用户偏好相匹配的物品。

　　混合推荐算法结合了协同过滤和内容推荐的优点，能够提供更准确的推荐结果。在本系统中，采用融合协同过滤和内容推荐的混合算法，综合考虑商品属性、用户偏好和时间因素。

2.4 安全防护技术

4.5 推荐算法设计

　　推荐算法是系统的核心功能之一，采用混合推荐策略，结合协同过滤和内容推荐两种方法，为用户提供个性化的商品推荐服务。

　　协同过滤算法采用基于物品的方法，通过计算商品之间的相似度进行推荐。相似度计算采用余弦相似度方法，考虑用户对商品的行为数据。为了考虑用户行为的时间特性，引入时间衰减因子，使得近期行为对推荐结果的影响更大，符合用户兴趣随时间变化的规律。

　　内容推荐算法根据商品的属性特征进行推荐，主要包括商品分类、价格区间、商品状况等特征。通过分析用户历史行为数据得出用户偏好，包括用户偏好的商品分类、价格区间等信息，然后根据商品属性与用户偏好的相似度进行推荐。

　　混合推荐策略将协同过滤和内容推荐的结果进行加权融合，综合考虑两种方法的推荐结果。协同过滤权重通常取0.6-0.7，内容推荐权重取0.3-0.4，通过调整权重参数，可以平衡两种推荐方法的影响。

　　针对新用户的冷启动问题，系统采用基于热门商品和商品属性的推荐策略。当用户行为数据不足时，推荐热门商品和基于商品分类的推荐；随着用户行为数据的积累，逐渐切换到个性化推荐。具体策略包括：新用户推荐热门商品和最新发布的商品；少量行为数据用户基于商品分类进行推荐；充足行为数据用户采用混合推荐算法进行个性化推荐。

5.2 个性化推荐算法实现

　　推荐算法采用混合推荐策略，结合协同过滤和内容推荐。

　　协同过滤部分采用基于物品的协同过滤算法，计算商品之间的相似度。相似度计算采用余弦相似度，考虑用户对商品的评分或行为。

　　内容推荐部分根据商品的属性特征进行推荐，包括商品分类、价格区间、品牌等。

　　混合策略将协同过滤和内容推荐的结果进行加权融合，综合考虑两种方法的推荐结果。

　　针对新用户的冷启动问题，系统采用基于热门商品和商品属性的推荐策略，随着用户行为数据的积累，逐渐切换到个性化推荐。

　　核心代码实现如下：

```python
# recommendation.py - 混合推荐算法核心实现
class HybridRecommender:
    """混合推荐算法 - 协同过滤 + 内容推荐"""
    
    def __init__(self):
        self.item_cf_weight = 0.6  # 协同过滤权重
        self.content_weight = 0.4  # 内容推荐权重
        self.time_decay_factor = 0.95  # 时间衰减因子
    
    def calculate_item_similarity(self, item_a, item_b):
        """计算两个商品的相似度（基于余弦相似度）"""
        # 获取对两个商品都有行为的用户
        users_a = set(UserBehavior.objects.filter(
            product=item_a, 
            behavior_type__in=['view', 'bookmark', 'transaction']
        ).values_list('user_id', flat=True))
        
        users_b = set(UserBehavior.objects.filter(
            product=item_b,
            behavior_type__in=['view', 'bookmark', 'transaction']
        ).values_list('user_id', flat=True))
        
        common_users = users_a & users_b
        
        if len(common_users) < 2:  # 共同用户太少，相似度为0
            return 0.0
        
        # 计算余弦相似度
        dot_product = len(common_users)
        norm_a = len(users_a) ** 0.5
        norm_b = len(users_b) ** 0.5
        
        similarity = dot_product / (norm_a * norm_b) if norm_a * norm_b > 0 else 0
        return similarity
    
    def get_item_based_recommendations(self, user, top_n=10):
        """基于物品的协同过滤推荐"""
        # 获取用户历史行为商品
        user_items = UserBehavior.objects.filter(
            user=user,
            behavior_type__in=['view', 'bookmark', 'transaction']
        ).select_related('product').distinct()
        
        if not user_items.exists():
            return []  # 冷启动情况
        
        # 计算候选商品与历史商品的相似度
        candidate_scores = {}
        user_item_ids = set(item.product_id for item in user_items)
        
        # 获取所有活跃商品作为候选
        candidate_items = Product.objects.filter(
            status='active'
        ).exclude(id__in=user_item_ids)
        
        for candidate in candidate_items[:100]:  # 限制候选数量
            total_score = 0
            for user_item in user_items:
                similarity = self.calculate_item_similarity(candidate, user_item.product)
                # 考虑时间衰减
                days_since = (timezone.now() - user_item.behavior_time).days
                time_weight = self.time_decay_factor ** days_since
                total_score += similarity * time_weight
            
            if total_score > 0:
                candidate_scores[candidate] = total_score
        
        # 返回Top-N推荐
        sorted_items = sorted(candidate_scores.items(), key=lambda x: x[1], reverse=True)
        return [item[0] for item in sorted_items[:top_n]]
```

5.3 安全防护模块实现

　　安全防护模块采用三层架构，包括身份安全、交易安全和系统安全。

　　身份安全通过校园邮箱认证实现，只允许特定教育域名的邮箱注册。同时提供密码强度检测功能，要求密码至少8位，包含大小写字母、数字和特殊字符。

　　交易安全通过异常行为检测机制实现，监控高频操作、价格异常、敏感词等风险行为。当检测到异常时，系统自动记录并发送安全预警。

　　系统安全通过输入验证和数据加密实现，对用户输入进行消毒处理，防止XSS攻击和SQL注入，对敏感数据进行加密存储。

　　核心代码实现如下：

```python
# security.py - 三层安全架构核心
class SecurityValidator:
    """安全验证器 - 三层安全架构核心"""
    
    @staticmethod
    def validate_email_domain(email):
        """身份安全：验证校园邮箱域名"""
        allowed_domains = ['edu.cn', 'edu']
        
        try:
            domain = email.split('@')[1].lower()
            for allowed_domain in allowed_domains:
                if domain.endswith(allowed_domain):
                    return True
            return False
        except:
            return False
    
    @staticmethod
    def validate_password_strength(password):
        """身份安全：验证密码强度"""
        if len(password) < 8:
            return False, "密码长度至少需要8位"
        
        if not re.search(r'[a-z]', password):
            return False, "密码需要包含小写字母"
        
        if not re.search(r'[A-Z]', password):
            return False, "密码需要包含大写字母"
        
        if not re.search(r'\d', password):
            return False, "密码需要包含数字"
        
        if not re.search(r'[!@#$%^&*(),.?":{}|<>]', password):
            return False, "密码需要包含特殊字符"
        
        return True, "密码强度合格"
    
    @staticmethod
    def detect_abnormal_behavior(user, action_type, action_data=None):
        """行为安全：检测异常行为模式"""
        if not user or not user.is_authenticated:
            return False
        
        is_abnormal = False
        risk_level = 'low'
        risk_description = ''
        
        # 检测短时间内高频操作
        recent_actions = UserBehavior.objects.filter(
            user=user,
            behavior_time__gte=timezone.now() - timedelta(minutes=5)
        ).count()
        
        if recent_actions > 50:
            is_abnormal = True
            risk_level = 'high'
            risk_description = f'5分钟内操作{recent_actions}次，可能存在机器人行为'
        
        # 检测价格异常
        if action_type in ['create_product', 'update_product'] and action_data:
            try:
                price = float(action_data.get('price', 0))
                if price > 100000:
                    is_abnormal = True
                    risk_level = 'medium'
                    risk_description = f'商品价格异常高: ￥{price}'
            except:
                pass
        
        # 检测敏感词
        if action_type in ['create_product', 'send_message'] and action_data:
            sensitive_words = ['赌博', '诈骗', '违禁品', '枪支', '毒品']
            content = str(action_data)
            for word in sensitive_words:
                if word in content:
                    is_abnormal = True
                    risk_level = 'high'
                    risk_description = f'内容包含敏感词: {word}'
                    break
        
        # 记录异常行为
        if is_abnormal:
            AbnormalBehavior.objects.create(
                user=user,
                behavior_type=f'abnormal_{action_type}',
                risk_level=risk_level,
                description=risk_description
            )
        
        return is_abnormal
    
    @staticmethod
    def sanitize_user_input(input_data):
        """数据安全：用户输入消毒"""
        if not input_data:
            return input_data
        
        input_str = str(input_data)
        
        # 移除危险标签和脚本
        dangerous_tags = [
            '<script>', '</script>', 
            '<iframe>', '</iframe>', 
            '<img', 'javascript:'
        ]
        for tag in dangerous_tags:
            input_str = input_str.replace(tag, '')
        
        # HTML实体编码
        input_str = input_str.replace('&', '&amp;')
        input_str = input_str.replace('<', '&lt;')
        input_str = input_str.replace('>', '&gt;')
        input_str = input_str.replace('"', '&quot;')
        input_str = input_str.replace("'", '&#x27;')
        
        return input_str

def calculate_cosine_similarity(vec_a, vec_b):
    """计算两个向量的余弦相似度"""
    dot_product = sum(a * b for a, b in zip(vec_a, vec_b))
    norm_a = sum(a * a for a in vec_a) ** 0.5
    norm_b = sum(b * b for b in vec_b) ** 0.5
    
    if norm_a * norm_b == 0:
        return 0.0
    
    return dot_product / (norm_a * norm_b)

class CollaborativeFiltering:
    """协同过滤推荐算法"""
    
    def __init__(self, user_item_matrix):
        self.user_item_matrix = user_item_matrix
        self.item_similarity_matrix = None
    
    def calculate_item_similarity_matrix(self):
        """计算物品相似度矩阵"""
        n_items = self.user_item_matrix.shape[1]
        self.item_similarity_matrix = np.zeros((n_items, n_items))
        
        for i in range(n_items):
            for j in range(i + 1, n_items):
                similarity = self.calculate_cosine_similarity(
                    self.user_item_matrix[:, i],
                    self.user_item_matrix[:, j]
                )
                self.item_similarity_matrix[i][j] = similarity
                self.item_similarity_matrix[j][i] = similarity
        
        return self.item_similarity_matrix
    
    def recommend(self, user_id, top_n=10):
        """为用户推荐商品"""
        user_ratings = self.user_item_matrix[user_id]
        scores = np.zeros(self.user_item_matrix.shape[1])
        
        for item_id in range(self.user_item_matrix.shape[1]):
            if user_ratings[item_id] == 0:  # 未评分的商品
                for rated_item in range(self.user_item_matrix.shape[1]):
                    if user_ratings[rated_item] > 0:
                        scores[item_id] += (
                            self.item_similarity_matrix[item_id][rated_item] * 
                            user_ratings[rated_item]
                        )
        
        # 返回Top-N推荐
        top_items = np.argsort(scores)[::-1][:top_n]
        return top_items
```
